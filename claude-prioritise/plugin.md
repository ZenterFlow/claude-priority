# Claude Prioritise

## Purpose
One-command project status assessment, priority surfacing, and persistent backlog grooming to help developers maintain context and focus across sessions.

## Activation Triggers
- "What should I work on?"
- "Show project status"
- "Update backlog"
- "/prioritise"

## Topics Covered
- Project status tracking (done, in progress, not started)
- Activity logging across sessions
- Backlog management and idea capture
- Priority surfacing based on context
- GitHub synchronization

## Workflow
1. Sync with GitHub for latest context
2. Load persistent logs, ideas, and todos
3. Process and prioritize tasks
4. Display current status and suggested next steps
5. Maintain activity log

## Resources
- Persistent storage in `persistence/` directory
- GitHub integration for up-to-date context
- Markdown-based logs for human readability
- Automatic backlog grooming
